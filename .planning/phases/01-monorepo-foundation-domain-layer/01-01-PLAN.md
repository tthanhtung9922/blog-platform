  <action>
Create `docker/init.sql` with PostgreSQL first-boot initialization:
```sql
-- docker/init.sql
-- Runs once on container first boot via docker-entrypoint-initdb.d
-- Creates the unaccent extension (idempotent — safe even if the EF Core migration also runs it)
CREATE EXTENSION IF NOT EXISTS unaccent;
```

Create `docker-compose.yml` at the repo root. CRITICAL rules:
- PostgreSQL 18 volume MUST target `/var/lib/postgresql` (NOT `/var/lib/postgresql/data` — that is the PG 16/17 path; PG 18 changed the internal PGDATA to `/var/lib/postgresql/18/docker`)
- MinIO init container uses `mc` (MinIO client) to create the `blog-media` bucket — this is the ONLY correct way (API startup code must not know about bucket initialization)
- Fixed ports, no .env needed for Phase 1

```yaml
services:
  postgres:
    image: postgres:18
    container_name: blog-postgres
    ports:
      - "5432:5432"
    environment:
      POSTGRES_USER: blog
      POSTGRES_PASSWORD: blog
      POSTGRES_DB: blog_db
    volumes:
      - postgres-data:/var/lib/postgresql
      - ./docker/init.sql:/docker-entrypoint-initdb.d/init.sql:ro
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U blog -d blog_db"]
      interval: 10s
      timeout: 5s
      retries: 5

  redis:
    image: redis:8
    container_name: blog-redis
    ports:
      - "6379:6379"
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5

  minio:
    image: quay.io/minio/minio:latest
    container_name: blog-minio
    ports:
      - "9000:9000"
      - "9001:9001"
    command: ["server", "--console-address", ":9001", "/data"]
    environment:
      MINIO_ROOT_USER: minio
      MINIO_ROOT_PASSWORD: minio123
    volumes:
      - minio-data:/data
    healthcheck:
      test: ["CMD", "mc", "ready", "local"]
      interval: 10s
      timeout: 5s
      retries: 5

  minio-init:
    image: quay.io/minio/mc:latest
    container_name: blog-minio-init
    depends_on:
      minio:
        condition: service_started
    restart: on-failure
    entrypoint: >
      /bin/sh -c "
      sleep 5;
      /usr/bin/mc alias set local http://minio:9000 minio minio123;
      /usr/bin/mc mb local/blog-media --ignore-existing;
      echo 'MinIO bucket blog-media ready.';
      exit 0;
      "

volumes:
  postgres-data:
  minio-data:
```

After writing both files, start the stack to verify:
```bash
docker-compose up -d postgres redis minio
sleep 10
docker-compose ps
```
